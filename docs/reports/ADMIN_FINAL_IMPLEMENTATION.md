## 執行決定

根據10位專家分析和PROWINE員工需求，採用**高效批量執行策略**：

### 策略
1. **重用現有表單結構** - 酒款/酒莊編輯頁面直接重用新增頁面的完整表單
2. **整合Toast系統** - 統一替換所有alert()為Toast通知
3. **補全缺失UI** - 添加keywords和SEO字段的UI界面
4. **創建詢價單API** - 添加狀態更新和回覆功能

1. 完成酒款編輯頁面（完整表單，重用新增頁面結構）

5. 批量替換所有alert()為Toast（32處）

### 關鍵技術點
- Toast使用：`import { useToast } from "@/components/admin/Toast"`
- 編輯模式：從API載入數據，使用PUT更新
- 完整字段：包括keywords、SEO等所有字段的UI
