四、執行步驟（直接給 Cursor 跑的 Roadmap）

建立參考說明檔

在專案中新增 REF_PROWINE_WEBSITES.md，貼上「給 Cursor 的總說明」＋上面 20 個網站表格。

要求 Cursor 逐一開啟連結，為每個網站輸出：hero 版型 / 導覽架構 / 主要模組 / 色彩與排版特徵 的簡表。

抽取共通 IA（資訊架構）

讓 Cursor 根據 B 類（10–18）先整理出一份「台灣酒商網站常見結構」：

例：首頁 / 關於我們 / 代理酒莊 / 酒款一覽 / 活動與課程 / 文章／知識 / 聯絡我們 / 會員或詢價區。

然後要求 Cursor 根據 ProWine 特色（小酒莊＋園丁＋教育）做一個「ProWine_IA_v1.md」。

決定視覺基調與 Design Tokens

指定 Cursor 以 A 類（2：Yquem、3：Silver Oak、6：Masi Tupungato、8：Artelium）為主參考，抽出：

色彩系統（深酒紅 / 炭黑 / 香檳金 / 暖米白）

字體階層（標題、內文、按鈕）

卡片風格（圓角、陰影、邊框）

產生 design-tokens-prowine.json 或 md 描述檔，之後所有頁面都吃同一套。

首頁與關鍵頁 Wireframe -> High-fidelity

讓 Cursor 先做 文字版 Wireframe，只標出區塊名稱與內容（不寫程式碼）：

首頁：Hero（品牌標語＋代表酒莊圖）、精選酒莊、精選酒款、學院內容、AI 選酒 CTA、聯絡。

代理酒莊列表頁：依產區／風格分群，每個酒莊卡片含照片＋風土標籤。

Wireframe 確定後，再請 Cursor 用剛剛的 Design Tokens 把 Wireframe 升級成 high-fidelity layout 描述。

功能模組設計（不急著上電商，可先做詢價／收藏）

參考 iCheers、Direct Wines Taiwan 的商品列表，先定義：

酒款卡片欄位（產區、品種、年份、價位區間、飲用場合 tag）。

篩選條件（產區、風格、價位、搭餐類型）。

結合 ProWine 的「顧問角色」，加上一個輕量版「AI 選酒」入口（僅前端 UI，日後再串 API）。

SEO 與文案框架

依照上述網站，要求 Cursor 生成：

每個主要頁面的 H1 / Meta Title / Meta Description 草稿。

針對「台北葡萄酒進口商、Napa 代理、精品葡萄酒推薦」等關鍵字規劃內容欄位。

五、風險評估

風險：設計太像國外某個精品酒莊，變成「抄襲感」

發生機率：中

影響：品牌形象受損、未來若要對外投資簡報會被挑戰。

對策：

讓 Cursor 明確標註「哪個元素來自哪一個網站」，再重新組合（混合 3–4 個來源）。

所有圖片改用 ProWine 自家酒莊照或付費圖庫，不直接抓對方圖。

風險：只學到「好看」，忽略 loading 與 RWD 實際表現

發生機率：中

影響：手機版速度慢，客戶實際使用體驗差。

對策：

在設計階段就要求 Cursor 給出「圖片壓縮策略、lazy load 策略」。

所有 hero 圖片設計兩套：desktop / mobile 版本。

風險：台灣使用者習慣與國外不同，轉換率不如預期

發生機率：中

影響：漂亮但不賣酒。

對策：

參考 iCheers、ENOTECA 的內容模組（FAQ、促銷區、活動）做 A/B 版本。
